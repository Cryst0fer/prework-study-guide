HTML

    HTML elements

    • All HTML elements must start with a !DOCTYPE declaration.
    • head - element contains information or metadata about the webpage which helps the browser to render the page correctly.
    • Elements within the head element are not visible to users looking at the webpage.
    • Within the head element, there can be additional tags that can be used such as:

      • Tags available in head element

      • style - element that defines the CSS styles associated with a website.
      • base - element that defines the base URL for a webpage.
      • link - element that connects an external resource to the HTML document.
      • meta - element that defines metadata such as the character set, description, keywords, author, and viewport.
      • script - element that embeds client-side scripts in the HTML document.

    • body - element contains the content that is shown to the user.
    • All elements that we want the user to interact with and see must be placed inside the body element.
    • In the example < html lang="en" > < html > is the tag and lang="en" is an attribute that defines additional information about the element.
    • The attribute lang="en" refers to the language for the website where "en" = English.
    • title - element defines the title for the webpage. This element is required and used for Search Engine Optimization (SEO).
    • Semantic HTML elements describes its content and the role it plays on a webpage. Example, < section >.
    • An element refers to what is inside the opening and closing tag. Example, "< h1>Panda Fan Site!< /h1>"
    • A tag refers to what is inside the angle brackets. Example, < "h1" >. Here the tag is h1.
    • h1...h6 represents the level of heading in a given text block. h1 is the largest
    • p - element that represents a paragraph or block of text.
    • ul - unordered list
    • ol - ordered list
    • li - list items
    • img - element that contains information about images displayed on the webpage. The src attribute defines the location of the image.
    • Example, < img src="https://example.webpage.com/image-01.jpg" >
    • br - element that creates a line of empty space or a line break between two blocks of content.
    • a - element that creates a link to the same webpage or other webpage which is also referred to as an anchor. The href attribute points to the URL in the anchor.
    • Example, < a href="https://example.webpage.com/some-image/image" >.
    • footer - element that contains the author, copyright, contact, sitemap, and navigation.
    • section - semantic element that defines a section in a webpage that contains thematic content.
    • id - attribute in < section > used as a way to identify elements and select them.
    • class - attribute in < section > ...undefined

GIT

    GitHub basics

    • repository - a folder for your project that contains all the files and their revision history.
    • branch - a separate workspace where changes do not affect the main branch.
    • markdown - the way content files are formatted in GitHub. A way to write for the web using plain text that is easily converted to HTML.
    • commit - saving a new version of a file you are working on whether it is an add, an edit, or a delete. The commit makes changes to the working branch.
    • pull request("PR") - asking for changes in your branch to be pulled or merged into another branch.
    • merge - when a pull request is accepted, changes are "pulled in" or merged into a branch.
    • issue - way to report new tasks or content fixes. Can be used to track progess on a board and labeling issues helps others to track it.

    GitHub flow

    1. Create a branch - create a short, descriptive branch name. Allows collaborators a chance to review your work and separate changes from the main branch.
    2. Make changes - make changes to your branch, then commit, and then push changes.
    3. Address review comments - make changes and commits until reviews/comments have all been addressed.
    4. Merge your pull request - one the pull request has been approved, merge the pull request to the main branch.
    5. Delete your branch - after merging the pull request, delete your branch to prevent you and others from re-using old branches.

    GitHub Terminal commands

      git checkout -b "branch": creates a new branch and switches to it.

      • git - must precede every Git statement
      • checkout - command to move the wokring branch to a new branch
      • -b - flag that creates a new branch
      • "branch" - name of the new branch

      git add -A

      • add - command that adds modifications in the current working branch to the staging area. The staging area informs Git changes will be in the next commit.
      • -A - flag that indicates that we want to add all the changes

      git pull origin main: used to check if any changes were made to the main branch to find conflicts before a merge

      • pull - command to receive another branch's modifications into the working branch
      • origin - keyword that indicates the source of the pull will be in the GitHub repo
      • main - keyword that indicates the branch to pull from

      git push origin "branch"

      • push - push the changes we have locally to our remote GitHub branch

      git checkout "branch": switches to the branch if not already in it

      git status: checks what branch we are currently in
